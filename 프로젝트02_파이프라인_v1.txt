8.시스템 프롬프트 핵심 규칙
	1. 제공된 근거에서만 답한다.
	2. 근거가 없는 내용은 추론하지 않는다.
	3. 금액·기한·수수료·대상·서류는 근거에 명시된 내용만 사용한다.
	4. 근거가 부족하면 부족하다고 명시한다.
	5. 답변에 사용한 출처 URL과 브레드크럼을 표시한다.
	6. 신청 질문이면 절차, 서류, 신청 페이지 순서로 안내한다.
	7. 서로 충돌하는 근거가 있으면 임의로 하나를 선택하지 않는다.

10. 평가 파이프라인 구현
검색 평가
Hit@K
정답 문서가 상위 K개 안에 하나라도 있는지 본다.
Hit@5 = 정답 문서가 Top-5에 포함되면 1
Recall@K
정답 문서가 여러 개일 때 얼마나 찾았는지 평가한다.
찾은 정답 문서 수 / 전체 정답 문서 수
MRR
첫 번째 정답이 몇 위에 나왔는지 평가한다.
1위면 1.0
2위면 0.5
3위면 0.333
타 업무 문서 혼입률
이번 프로젝트에서 매우 중요한 지표다.
Top-K 검색 결과 중 다른 업무 청크 수 / Top-K
단, 필터를 걸면 항상 0이 될 수 있으므로 이것만 봐서는 안 된다. 잘못된 업무 분류로 정답 문서를 차단하지 않았는지도 함께 평가해야 한다.

답변 평가
최소 다음 지표가 필요하다.

지표	          		 의미
Correctness 			공식 정답과 내용이 맞는가
Completeness		필수 사실이 빠지지 않았는가
Faithfulness	   		답변이 검색 근거에 존재하는가
Source Accuracy		올바른 출처를 표시했는가
Citation Coverage		주요 주장마다 근거가 있는가
Action Link Accuracy	올바른 신청·서류 링크를 제공했는가
Abstention Accuracy	답변 불가 질문에 제대로 답변을 거절했는가
